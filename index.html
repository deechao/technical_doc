    • the basic structure of HTML

    Before you can start adding content to your document, there's a basic structure you need to set up in your file. This structure isn't only required for it to work but will also you to provide useful information about your document. The basic structure of any HTML document consists of the following sections or elements.

  • ·The DTD !DOCTYPE declaration
  • ·The main container html element
  • ·The head section head element
  • ·The body section body element

    • HTML declaration

    Every HTML document must begin with a basic declaration that identifies it's type. This is a very useful measure that inform browsers beforehand, what type of document they are about to process, allowing them to adjust their processing mechanisms accordingly.

    The DTD is insterted using a special tag !DOCTYPE that takes a particular form for each type. This declaration can only be present at the beginning of the document. The following example shows the DTD for an HTML5 document.

    <!DOCTYPE html>

    the HTML element

    After having placed the DTD right at the top of the document, it is time to create the main container: a place where the whole document (minus the DTD) will fall into. This container is insterted with the <html> element, it provides a good chance to define the default language used by the document through the global attribute lang.

    The following example shows the structure of the main container (including the language used in the document) and indicates where all the elements of the document should be placed.

    <html>

    </html>

    The content of this element can be further divided into two sections: the <head> and the <body>.

    the head element

    The <head> section is a container for metadata about the document. This metadata can be classified in five categories according to the element used.

  • The document's title:describes briefly the subject treated in the document. This is a required item and is inserted with the <title> element
  • Style declarations:groups style definitions used to set presentational attributes for the elements in the document. It's inserted with the <style> element
  • Client-side scripts:inserts programs that provide functionality and interactivity. It's declared with the <script> element
  • Meta statements:define custom attributes and values. They're inserted with the <meta> element
  • Relational information:ndicates resources that are somehow related to the document. It's inserted with the <link> element

    the body element

    The <body> of a document is basically the container for its rendered part. This is the area where you start writing your content which is the part the viewers will see. There are many types of html tags that can go here and with practice, you will be able to know the basics that are commonly used for a more clean structure.
